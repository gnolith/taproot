# Governance

Taproot is currently maintained by `@kcsfelty` under the Gnolith organization.
The maintainer decides releases, compatibility boundaries, security response,
and repository policy after considering public issue and pull-request input.

Substantial changes should begin with an issue. Compatibility claims require
fixtures or behavioral tests. Security-sensitive reports follow `SECURITY.md`.
Routine changes use pull requests and the protected `main` branch; releases
must pass the repository release workflow.

As the contributor base grows, maintainership may be granted based on sustained
technical contributions, sound review judgment, and reliable stewardship.
Changes to governance are made publicly through pull requests.
